# booping — /help

Orient the user to the booping plugin. This skill is read-only — it never scaffolds, migrates, or writes files. That is `/install`'s job.

## Preflight

- Read and resolve project based on [project resolution principle](../../docs/partial_project_resolution.md) — detect whether the current CWD is inside an attached repo so the Quickstart block can personalize "first time in a repo" guidance.

## High-level workflow

1. Dispatch topic based on `$ARGUMENTS`.
2. Render body.

## Phase 0 — Dispatch topic

Decide which body section(s) to print based on `$ARGUMENTS`:

- Empty → print the **Quickstart** block only.
- `skills`, `agents`, `layout`, `workflow` → print the matching inline section only.
- Anything else → print the full document (all inline sections).

## Phase 1 — Render body

Print the section(s) chosen in Phase 0 verbatim, in this order: Quickstart → Skills → Agents → Layout → Workflow → Hard rules → See also.

### Quickstart

```
booping — per-project grooming / implementation / retro / lessons with mandatory sub-agent delegation

First time in a repo:    /install
Start a discussion:      /chat
Spec new work:           /groom <topic>
Execute a plan:          /develop ~/Claude/<project>/plans/YYYYMMDD-*.md
After a sprint ships:    /retro ~/Claude/<project>/plans/YYYYMMDD-*.md
Extract lessons:         /learn [~/Claude/<project>/retrospectives/YYYYMMDD-*.md]
```

Current project resolution: `.booping` marker in CWD → ask.
Artifacts live at `~/Claude/{project}/`. `sprints.md` is regenerated by `/chat` via `booping-plans --format=md`.

### Skills

| Slash | Purpose | Writes |
|-------|---------|--------|
| `/install` | Scaffold a new booping project and/or attach the current repo. | `~/Claude/{project}/*`, `.booping` |
| `/help` | Orient the user; read-only tour of the plugin. | — |
| `/chat` | Discussion mode with vault orientation. Regenerates `sprints.md` snapshot on orient. | `sprints.md` |
| `/groom` | Deep-research a feature/bug/refactor into `plans/YYYYMMDD-*.md`; saves the request. | `requests/`, `plans/` |
| `/develop` | Execute a groomed plan milestone-by-milestone; always delegates to sub-agents. | plan progress marks (DoD + status) |
| `/retro` | Review what shipped, gather feedback, produce retrospective. Transitions `awaiting-retro → awaiting-learning`. | `retrospectives/` |
| `/learn` | Extract lessons from a retrospective; presents a unified review table; user accepts/rejects/adds. Transitions `awaiting-learning → done`. | `lessons/`, `_booping/*.md` |

### Agents

**Researchers** (delegated to from skills for read-heavy work):

- `booping-researcher-junior` — narrow lookups (haiku tier).
- `booping-researcher-middle` — default research (sonnet).
- `booping-researcher-senior` — hard / cross-cutting research (opus, reasoning high).

Tier is selected by the skill per `docs/partial_agents_researchers_delegator.md`.

**Developers** (called from `/develop`):

- `booping-developer-middle` — 1–2 SP tasks; may be batched (sonnet).
- `booping-developer-senior` — 3–4 SP design-judgment tasks; one per briefing (opus, reasoning high).

5 SP tasks are refused — kicked back to `/groom` for re-decomposition. Active SP→agent mapping lives in `docs/partial_agents_strategy_mid_senior.md`.

### Layout

```
~/Claude/
└── {project}/
    ├── plans/                   # /groom output
    ├── requests/                # /groom input (saved user requests)
    ├── retrospectives/          # /retro output
    ├── lessons/                 # /learn output
    ├── _booping/                # project-local skill/agent extensions
    └── sprints.md               # regenerated by /chat
```

### Workflow

```
/chat   ──► vault orientation; regenerates sprints.md via booping-plans --format=md

/groom  ──► requests/YYYYMMDD-*.md  →  plans/YYYYMMDD-*.md

/groom  ──► backlog → in-spec → awaiting-plan-review → ready-for-dev
            (on user approval; cancelled / back-to-in-spec branches otherwise)

/develop ──► sprint branch; per milestone:
             • spawns booping-developer-{middle,senior} per task
             • milestone-diff review via booping-researcher-{middle,senior}
             • flips DoD + milestone status in plan frontmatter
             transitions plan: ready-for-dev → in-progress → awaiting-retro

/retro  ──► delegates session-log search to booping-researcher-middle
            orchestrator synthesizes → retrospectives/YYYYMMDD-*.md
            transitions plan: awaiting-retro → awaiting-learning

/learn  ──► unified review table (accept/reject/add)
            lessons/{N}_*.md + _booping/skill_*.md + _booping/agent_*.md
            transitions plan: awaiting-learning → done
```

### Hard rules

- **`/help` never writes files.** If the user needs scaffolding, hand off to `/install`.
